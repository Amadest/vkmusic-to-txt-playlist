## Как это работает

Нормальный сценарий переноса музыки между сервисами выглядит так:

1. Собираем треки из исходного сервиса в TXT в формате `Исполнитель - Название`.
2. Если файл большой, режем его на части по 500 строк.
3. Загружаем эти TXT в TuneMyMusic:
   [tunemymusic.com/transfer/text-file-to-spotify](https://www.tunemymusic.com/transfer/text-file-to-spotify)
4. Получаем на целевой платформе обычные плейлисты.
5. Если нужно перенести именно избранные треки в Spotify, используем отдельный браузерный скрипт: он идёт по плейлисту, ставит лайк каждому треку и удаляет его из плейлиста.

Почему так:

- плейлисты между сервисами перегоняются легко
- избранное обычно перегоняется плохо или вообще не перегоняется напрямую
- поэтому избранное сначала превращаем в обычный плейлист
- потом в Spotify догоняем его до `Liked Songs` через браузерную автоматизацию

## Экспорт из VK

### Если просто запустить

1. Открой PowerShell в папке проекта
2. Для обычного плейлиста вставь ссылку и запусти:

```bash
npm run export -- --playlist "ССЫЛКА_НА_ПЛЕЙЛИСТ_VK"
```

3. Для `Моя музыка` запусти:

```bash
npm run export -- --playlist my-music
```

4. Если откроется браузер и попросит логин, просто войди в VK
5. Готовый TXT появится в папке `playlists/`

### В чём смысл

Для VK есть два основных источника:

- обычные плейлисты по ссылке
- раздел `Моя музыка`

На выходе получаем TXT, который потом можно загрузить в TuneMyMusic и перегнать в нужный сервис по 500 строк за раз.

### Как это работает

Есть два варианта:

- CLI-режим: скрипт сам открывает браузер, ждёт логин и собирает треки
- `F12 -> Console`: скрипт печатается в консоль, ты вставляешь его вручную на странице VK

CLI:

```bash
npm run export -- --playlist "https://vk.com/music/playlist/123_456_hash"
npm run export -- --playlist my-music
```

Ручной вариант:

```bash
node src/cli.js snippet
```

Если Chrome попросит подтверждение вставки кода в DevTools, сначала нужно ввести:

```text
allow pasting
```

Что дальше:

1. Получаешь TXT в папке `playlists/`
2. Если нужно, режешь его по 500 строк
3. Загружаешь части в TuneMyMusic

Разбиение:

```bash
npm run split -- --path "./playlists/Моя музыка.txt" --max-lines 500
```

## Экспорт из Яндекс.Музыки

### Если просто запустить

1. Сделай нужный плейлист публичным
2. Скопируй ссылку на него
3. Запусти:

```bash
npm run yandex-export -- --playlist "ССЫЛКА_НА_ПЛЕЙЛИСТ_ЯНДЕКСА"
```

4. Если файл большой и нужен сразу по 500 строк:

```bash
npm run yandex-export -- --playlist "ССЫЛКА_НА_ПЛЕЙЛИСТ_ЯНДЕКСА" --split --max-lines 500
```

5. Готовый TXT появится в `playlists/`, а части по 500 в `split/`

### В чём смысл

Для Яндекс.Музыки логика та же самая: нужно получить обычный TXT-плейлист, который потом можно отдать в TuneMyMusic.

Если плейлист публичный, авторизация в Яндекс.Музыке вообще не нужна. Скрипт может открыть чистую браузерную сессию без вашего профиля, куков и личных данных и просто собрать треки по публичной ссылке.

### Обычные плейлисты

Экспорт обычного плейлиста:

```bash
npm run yandex-export -- --playlist "https://music.yandex.ru/playlists/..."
```

### Плейлист `Мне нравится`

С `Мне нравится` проще всего не воевать отдельно, а сделать этот плейлист публичным и экспортировать его как обычный плейлист по ссылке.

То есть схема такая:

1. Делаешь плейлист `Мне нравится` публичным в Яндекс.Музыке
2. Берёшь его публичную ссылку вида `https://music.yandex.ru/playlists/lk....`
3. Выгружаешь как обычный плейлист
4. При необходимости сразу режешь на части по 500 строк

Это удобно ещё и потому, что после публикации не нужна авторизация: можно просто открыть ссылку в чистой сессии браузера и выгрузить треки как у любого обычного публичного плейлиста.

Команда:

```bash
npm run yandex-export -- --playlist "https://music.yandex.ru/playlists/lk...." --split --max-lines 500
```

### Ручной вариант

Если удобнее через браузерную консоль:

```bash
node src/cli.js yandex-snippet
```

## Spotify: как добить избранное

### Если просто запустить

1. Открой Chrome с remote debugging:

```bash
& "C:\Program Files\Google\Chrome\Application\chrome.exe" --remote-debugging-port=9222
```

2. В этом Chrome открой нужный плейлист в Spotify
3. Запусти в PowerShell:

```bash
node src/cli.js spotify-like-attach --remove-from-playlist
```

4. Скрипт сам пройдёт по плейлисту, поставит лайки и будет удалять обработанные треки
5. В конце треки окажутся в `Liked Songs`

### Зачем вообще нужен этот скрипт

Это костыль для переноса избранного в Spotify.

Проблема в том, что сторонние сервисы нормально переносят плейлисты, но не всегда умеют адекватно переносить `Liked Songs`.

Поэтому рабочая схема такая:

1. В исходном сервисе собираем избранное как обычный плейлист
2. Через TuneMyMusic переносим его в Spotify как обычный плейлист
3. Отдельным скриптом проходим по этому плейлисту в Spotify
4. Скрипт ставит лайк каждому треку
5. После лайка удаляет трек из плейлиста

На выходе:

- плейлист постепенно схлопывается
- треки оказываются в `Liked Songs`
- не нужно бороться с виртуальным скроллом огромного плейлиста

### Как запускать

Открыть Chrome с remote debugging:

```bash
& "C:\Program Files\Google\Chrome\Application\chrome.exe" --remote-debugging-port=9222
```

Открыть в этом Chrome нужный Spotify-плейлист и запустить:

```bash
node src/cli.js spotify-like-attach --remove-from-playlist
```

Флаг `--remove-from-playlist` рекомендуется всегда: именно он делает сценарий удобным для переноса избранного.

### Что умеет скрипт

- ставит лайки трекам из открытого Spotify-плейлиста
- может удалять трек из плейлиста сразу после лайка
- сохраняет JSON-отчёт
- умеет повторять пропущенные треки из предыдущего отчёта

## Быстрый старт

### VK -> Spotify

1. Выгрузить плейлист VK или `Моя музыка` в TXT
2. При необходимости разбить TXT на части по 500 строк
3. Импортировать части через TuneMyMusic
4. Если это был плейлист для избранного, открыть его в Spotify и прогнать `spotify-like-attach --remove-from-playlist`

### Яндекс.Музыка -> Spotify

1. Выгрузить плейлист Яндекс.Музыки в TXT
2. Для `Мне нравится` сначала сделать плейлист публичным
3. При необходимости разбить TXT на части по 500 строк
4. Импортировать части через TuneMyMusic
5. Если это плейлист для будущего избранного Spotify, прогнать `spotify-like-attach --remove-from-playlist`

## Команды

### `export`

```bash
npm run export -- --playlist "https://vk.com/music/playlist/123_456_hash"
npm run export -- --playlist my-music
```

Параметры:

- `--playlist` — ссылка на плейлист VK или `my-music`
- `--browser` — `chrome`, `edge` или `firefox`
- `--out` — путь для сохранения TXT
- `--profile-dir` — директория managed session
- `--executable-path` — путь к бинарнику браузера
- `--attach` — подключиться к уже открытому Chrome/Edge через `http://127.0.0.1:9222`
- `--attach-url` — свой remote debugging endpoint
- `--headless` — запуск без видимого окна

### `yandex-export`

```bash
npm run yandex-export -- --playlist "https://music.yandex.ru/playlists/..."
npm run yandex-export -- --playlist "https://music.yandex.ru/playlists/lk...." --split --max-lines 500
```

Параметры:

- `--playlist` — ссылка на плейлист Яндекс.Музыки
- `--browser` — `chrome`, `edge` или `firefox`
- `--out` — путь для сохранения TXT
- `--attach` — подключиться к уже открытому Chrome/Edge через `http://127.0.0.1:9222`
- `--attach-url` — свой remote debugging endpoint
- `--split` — сразу записать части по несколько файлов
- `--max-lines` — размер части для `--split`, по умолчанию 500
- `--split-out-dir` — своя папка для частей
- `--profile-dir` — директория managed session
- `--executable-path` — путь к бинарнику браузера
- `--headless` — запуск без видимого окна

### `validate`

```bash
npm run validate -- --path "./playlists/Моя музыка.txt"
```

Проверяет формат TXT и показывает статистику.

### `split`

```bash
npm run split -- --path "./playlists/Моя музыка.txt" --max-lines 500
```

Разбивает большой TXT на части. По умолчанию 500 строк на файл.

### `snippet`

```bash
node src/cli.js snippet
```

Печатает JS-сниппет для ручного запуска через `F12 -> Console` на странице VK.

Если DevTools блокирует вставку, сначала введи `allow pasting`, а потом уже вставляй сниппет.

### `yandex-snippet`

```bash
node src/cli.js yandex-snippet
```

Печатает JS-сниппет для ручного запуска через `F12 -> Console` на странице Яндекс.Музыки.

### `spotify-like-snippet`

```bash
node src/cli.js spotify-like-snippet
```

Печатает JS-сниппет для ручного лайка треков прямо из консоли Spotify.

### `spotify-like-attach`

```bash
node src/cli.js spotify-like-attach [опции]
```

Подключается к открытому Chrome и ставит лайки на треки из открытого Spotify-плейлиста.

Параметры:

- `--attach-url` — remote debugging endpoint, по умолчанию `http://127.0.0.1:9222`
- `--remove-from-playlist` — удалять трек из плейлиста после лайка
- `--max-new-likes` — лимит новых лайков за запуск
- `--retry-per-row` — попыток на трек, по умолчанию 5
- `--retry-skipped` — повторить пропущенные треки в конце того же запуска
- `--report` — путь для JSON-отчёта

### `spotify-like-attach-retry`

```bash
node src/cli.js spotify-like-attach-retry --from-report <путь> [опции]
```

Повторяет только треки со статусом `menu-not-found` из предыдущего отчёта.

### `liked-sync`

```bash
node src/cli.js liked-sync --path "./playlists/Моя музыка.txt" --spotify-client-id <id>
```

API-вариант для Spotify. Работает медленнее и упирается в rate limit, поэтому основной рабочий путь здесь всё равно браузерный.

### `liked-sync-playlist`

```bash
node src/cli.js liked-sync-playlist --playlist <spotify-url|id> --spotify-client-id <id>
```

То же, но берёт треки напрямую из Spotify-плейлиста.

## Отчёты

`spotify-like-attach` сохраняет JSON-отчёт в папку `reports/`.

Каждая запись содержит:

- `key` — track ID из Spotify
- `label` — текст строки как он отображается в UI
- `status` — `liked`, `already-liked`, `action-not-found`, `menu-not-found`

По этому отчёту можно запускать повтор через `spotify-like-attach-retry`.

## Безопасность

Весь процесс локальный:

- экспорт из VK и Яндекс.Музыки выполняется на вашем устройстве
- attach-режим подключается только к браузеру, который уже запущен локально у вас
- сессии хранятся локально в `.session/`
- TXT и отчёты сохраняются локально в проекте
- логины и пароли не передаются в сторонний сервис этим инструментом

Для публичных плейлистов Яндекс.Музыки можно вообще не логиниться: достаточно открыть ссылку в чистой браузерной сессии и собрать TXT без использования вашего аккаунта.

## Платформы

| ОС      | Статус |
| ------- | ------ |
| Windows | основной рабочий сценарий, проверен |
| macOS   | поддержан в коде |
| Linux   | поддержан в коде |

Важно:

- `F12 -> Console` не зависит от ОС и работает везде
- attach-режим работает только для Chromium-браузеров, таких как Chrome и Edge
- Firefox лучше использовать через managed session или `snippet`

## Ограничения

- VK периодически меняет разметку, поэтому парсер может требовать обновлений
- для `Моя музыка` VK может показывать только видимую часть библиотеки за подпиской
- для `Мне нравится` в Яндекс.Музыке проще использовать публичную ссылку на плейлист
- бесплатный TuneMyMusic обычно означает лимит 500 треков за перенос
- Spotify API в режиме разработки имеет жёсткий rate limit и хуже подходит для больших библиотек
